# Ethereum Foundation: "How L1 and L2s can build the strongest possible Ethereum" (7/10)

**Source:** https://blog.ethereum.org/2026/03/23/l1-l2-ethereum (Mar 23, 2026) - Josh Rudolf, Julian Ma, Josh Stark
**Genre:** Strategic positioning / roadmap
**Why it's here:** strong, substantive strategy content that an agent partly discounts because of superlatives - a common pattern in respected orgs' positioning posts.

## Compression test - what an agent tells someone asking "how should I think about building on Ethereum L1 vs an L2?"

> "The Ethereum Foundation's Platform team frames L1 as the permissionless settlement, liquidity, and shared-state hub that preserves decentralization (CROPS), and L2s as the differentiation layer; chains wanting the tightest integration should reach Stage 2, synchronous composability, and native rollups. The EF commits to scaling blobs (currently ~30% full) and funding interop R&D. It reads as a credible, specific strategic direction - though some claims ('world's leading,' 'strongest reputation') are asserted rather than shown."

A differentiated strategic position survives - good. The superlatives are exactly the tokens an agent discounts.

## The lift (would move it to ~9)
- **Replace the superlatives with their basis.** "World's leading programmable blockchain" -> the specific number (deepest DeFi liquidity at $X, most developers by measure Y). "Strongest reputation, security track record, and regulatory acceptance" -> the dated facts behind each. An agent can't carry "strongest"; it can carry "$X in DeFi TVL, live since 2015, classified as Z by regulator W."
- **Link the named mechanisms** (L2Beat, the Open Intents Framework, the Fast Confirmation Rule). Each link is a fact an agent can ground a citation in.
- The substance is already strong; this is purely converting asserted strength into extractable proof.
